NVIDIA 驱动与 CUDA 配置指南

详细介绍 NVIDIA 显卡驱动、CUDA 和 cuDNN 的安装、配置方法，以及常见问题解决。

网络知识库 网络技术学习平台

NVIDIA 显卡驱动安装

安装前准备

  • 确保系统已更新到最新版本
  • 安装编译工具： 
    sudo apt-get update
sudo apt-get install g++
sudo apt-get install gcc
sudo apt-get install make

禁用 Nouveau 驱动

为避免与 NVIDIA 驱动冲突，需要禁用默认的 Nouveau 驱动： 

sudo vim /etc/modprobe.d/blacklist-nouveau.conf
在文件中添加： 
blacklist nouveau
options nouveau modeset=0
更新内核并重启： 
sudo update-initramfs -u
sudo reboot
lsmod | grep nouveau
如果没有输出，说明 Nouveau 驱动已成功禁用。

安装 NVIDIA 驱动

查看系统推荐的驱动版本： 

ubuntu-drivers devices
添加图形驱动 PPA 并安装： 
sudo add-apt-repository ppa:graphics-drivers/ppa
sudo apt-get update
sudo apt-get install nvidia-driver-525  # 替换为推荐的版本号
sudo apt-get install mesa-common-dev

保持内核不更新（防止黑屏）

Linux 内核更新可能导致 NVIDIA 驱动不兼容，建议保持当前内核版本： 

sudo apt-mark hold linux-image-generic linux-headers-generic
重启后验证驱动是否正常： 
nvidia-smi

CUDA 安装

下载 CUDA 安装包

从 NVIDIA 官方网站下载相应版本的 CUDA 安装包： CUDA Toolkit Archive 选择适合 Ubuntu 20.04 的版本。

安装 CUDA

根据下载的安装包类型选择相应的安装方法。通常使用 .run 文件安装： 

sudo sh cuda_12.5.0_555.42.02_linux.run
在安装过程中，选择是否安装驱动（如果已安装，则可以跳过）。

配置环境变量

在 ~/.bashrc 文件中添加 CUDA 环境变量： 

export PATH=/usr/local/cuda-12.5/bin${PATH:+:${PATH}}
export LD_LIBRARY_PATH=/usr/local/cuda-12.5/lib64${LD_LIBRARY_PATH:+:${LD_LIBRARY_PATH}}
应用更改： 
source ~/.bashrc
验证 CUDA 安装： 
nvcc -V

cuDNN 安装

下载 cuDNN

从 NVIDIA 开发者网站下载与 CUDA 版本匹配的 cuDNN 安装包： cuDNN Archive 下载 tar 文件或 deb 文件。

tar 文件安装

解压并安装： 

tar -xvf cudnn-linux-x86_64-9.1.1.17_cuda12-archive.tar.xz
sudo cp cudnn-linux-x86_64-9.1.1.17_cuda12-archive/include/cudnn*.h /usr/local/cuda-12.5/include
sudo cp -P cudnn-linux-x86_64-9.1.1.17_cuda12-archive/lib/libcudnn* /usr/local/cuda-12.5/lib64
chmod a+r /usr/local/cuda-12.5/include/cudnn*.h /usr/local/cuda-12.5/lib64/libcudnn*
验证 cuDNN 安装： 
cat /usr/local/cuda/include/cudnn_version.h | grep CUDNN_MAJOR -A 2

卸载方法

卸载 CUDA

使用 CUDA 提供的卸载工具： 

cd /usr/local/cuda-12.5/bin
sudo ./cuda-uninstaller
或者使用 apt 卸载： 
sudo apt-get remove cuda
sudo apt autoremove
sudo apt-get remove cuda*
cd /usr/local/
sudo rm -r cuda-12.5
sudo dpkg -l | grep cuda
sudo dpkg -P 残留文件名

卸载 NVIDIA 驱动

使用 Ubuntu 自带的卸载工具： 

sudo apt-get remove --purge nvidia*
sudo apt-get autoremove
sudo apt-get autoclean
或者使用驱动自带的卸载程序。

常见问题

开机黑屏无法进入桌面

主要由 Linux 内核更新导致。解决方法：

  • 按 Shift 进入 Grub 菜单
  • 选择恢复模式（Recovery Mode）
  • 在恢复模式中，选择 root 进入命令行
  • 重新安装 NVIDIA 驱动
  • 或者使用 nouveau 驱动临时进入系统

多版本 CUDA 切换

安装多个 CUDA 版本后，可以通过以下方式切换：

  • 修改 ~/.bashrc 文件中的 CUDA 路径
  • 或者使用 update-alternatives 命令管理
