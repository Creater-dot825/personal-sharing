Docker 安装与配置指南

详细介绍 Docker 在 Ubuntu 系统上的安装方法，包括官方源和国内镜像源，以及常见配置。

网络知识库 网络技术学习平台

Docker 安装

卸载旧版本

如果之前安装过 Docker，需要先卸载旧版本： 

sudo apt-get remove docker docker-engine docker.io containerd runc

安装依赖包

安装 Docker 所需的依赖包： 

sudo apt install apt-transport-https ca-certificates curl software-properties-common gnupg lsb-release

添加 Docker GPG 密钥

官方源 GPG 密钥

curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg

阿里源 GPG 密钥（推荐国内用户）

curl -fsSL https://mirrors.aliyun.com/docker-ce/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg

添加软件仓库

官方源

echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

阿里源（推荐国内用户）

echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://mirrors.aliyun.com/docker-ce/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

更新源并安装 Docker

sudo apt update
sudo apt-get update

# 安装最新版本的 Docker
sudo apt install docker-ce docker-ce-cli containerd.io -y

验证安装

# 查看 Docker 版本
sudo docker version

# 查看 Docker 运行状态
sudo systemctl status docker

Docker 配置

启用 Docker Bash 自动补全

sudo apt-get install bash-completion
sudo curl -L https://raw.githubusercontent.com/docker/docker-ce/master/components/cli/contrib/completion/bash/docker -o /etc/bash_completion.d/docker.sh
source /etc/bash_completion.d/docker.sh

允许非 root 用户运行 Docker

sudo groupadd docker
sudo usermod -aG docker $USER
newgrp docker
docker ps -a
为了避免每次打开新终端都需要执行 `newgrp docker`，需要在 `~/.bashrc` 文件末尾添加： 
groupadd -f docker

Docker 常用命令

  • docker run [image]：运行容器
  • docker ps：查看运行中的容器
  • docker ps -a：查看所有容器
  • docker stop [container_id]：停止容器
  • docker start [container_id]：启动容器
  • docker rm [container_id]：删除容器
  • docker images：查看本地镜像
  • docker rmi [image_id]：删除镜像
  • docker pull [image]：拉取镜像

修改 Docker 镜像源

为了提高镜像下载速度，建议使用国内镜像源。修改 `/etc/docker/daemon.json` 文件（如果不存在则创建）： 

sudo tee /etc/docker/daemon.json <<-'EOF'
{
  "registry-mirrors": ["https://docker.mirrors.ustc.edu.cn", "https://hub-mirror.c.163.com"]
}
EOF
重启 Docker 服务： 
sudo systemctl daemon-reload
sudo systemctl restart docker
验证是否生效： 
docker info
在输出中会看到 "Registry Mirrors" 信息。

常见问题

Docker 启动失败

  • 检查 Docker 服务状态：systemctl status docker
  • 查看日志：journalctl -xeu docker
  • 检查网络连接
  • 确保 /var/lib/docker 目录有足够权限

非 root 用户无法运行 Docker

  • 确保用户已加入 docker 组
  • 检查 groupadd 和 usermod 命令是否成功
  • 尝试重新登录或重启系统
